Architecture Decisions

  • ✅ EKS vs Self-managed: Managed control plane

AWS Design Summary

High Availability Architecture

  • VPC: Multi-AZ (10.0.0.0/16)
  • Compute: EKS nodes across 2 AZs
  • Database: RDS Multi-AZ + backups
  • Cache: ElastiCache with failover
  • Load Balancing: ALB with health checks
  • Networking: NAT, Security Groups

Infrastructure as Code

  • Terragrunt: DRY configuration
  • Remote State: S3 + DynamoDB locking
  • Modules: VPC, EKS, IAM, RDS
  • Validation: Input constraints

Internet → ALB → EKS Cluster
               ├── Frontend Pods
               ├── Backend Pods
               ├── RDS (Multi-AZ)
               └── ElastiCache
